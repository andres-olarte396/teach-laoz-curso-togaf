**Tiempo estimado**: 35 minutos
**Nivel**: Avanzado
**Prerrequisitos**: Definicion del Escenario (10.1.1)

## Por que importa este subtema?

Antes de iniciar el ciclo ADM, es fundamental entender los requisitos del negocio y las restricciones que limitaran las opciones de arquitectura. Estos elementos guian todas las decisiones arquitectonicas.

## Restricciones

### Restricciones de Presupuesto

```markdown
## Budget Constraints

### Inversion Total
| Concepto | Monto | Periodo |
|----------|-------|---------|
| CAPEX Transformacion | $25M | 3 anos |
| OPEX Adicional (cloud) | $8M/ano | Post-go-live |
| Contingencia | 15% | Incluido |

### Distribucion por Ano
| Ano | Inversion | Foco |
|-----|-----------|------|
| 2026 | $10M | Fundamentos, ecommerce |
| 2027 | $10M | Tiendas, supply chain |
| 2028 | $5M | Optimizacion, analytics |

### Restricciones
- ROI positivo en 30 meses
- No superar budget anual aprobado
- Reducir OPEX legacy como se migra
```

### Restricciones Tecnologicas

```markdown
## Technical Constraints

### Mandatorias
| Restriccion | Razon |
|-------------|-------|
| AWS como cloud primario | Contrato existente, skills |
| SAP S/4HANA para ERP | Inversion existente en SAP |
| No desarrollo custom de core commerce | Buy vs build |
| Kubernetes para contenedores | Standard corporativo |
| PostgreSQL o Aurora para nuevas DBs | Standard |

### Legacy a Mantener
| Sistema | Razon | Hasta Cuando |
|---------|-------|--------------|
| SAP ECC | Migrar a S/4 | 2027 |
| Oracle WMS | Evaluar reemplazo | 2028 |
| Mainframe batch | Migrar gradualmente | 2027 |

### Prohibiciones
| Restriccion | Razon |
|-------------|-------|
| No Oracle Cloud | Evitar lock-in adicional |
| No servidores fisicos nuevos | Cloud-first |
| No desarrollo en ABAP | Modernizacion |
```

### Restricciones Temporales

```markdown
## Timeline Constraints

### Fechas Criticas
| Milestone | Fecha | Importancia |
|-----------|-------|-------------|
| Black Friday 2026 | Nov 2026 | Nuevo ecommerce debe estar live |
| Cierre fiscal | Dic 2026 | Sin migraciones ERP en Q4 |
| Expansion regional | Q1 2028 | Arquitectura debe soportar |
| SAP ECC EOL | 2027 | Migrar antes |
| Lease datacenter | Jun 2028 | Salir de DC actual |

### Restricciones de Implementacion
- No deployments en produccion viernes-domingo
- Freeze de cambios en temporada alta (Nov-Dic)
- Migraciones de datos solo fines de semana
- 6 meses minimo para capacitacion antes de go-live
```

### Restricciones Organizacionales

```markdown
## Organizational Constraints

### Recursos
| Restriccion | Impacto |
|-------------|---------|
| Equipo TI de 120 personas | Capacidad limitada |
| Skills cloud limitados | Necesita capacitacion |
| Resistencia al cambio | Gestion del cambio critica |
| Dependencia de vendors | SAP, Oracle expertise |

### Politicas
| Politica | Requisito |
|----------|-----------|
| Outsourcing limite | Max 40% del equipo externo |
| Trabajo remoto | Equipos pueden ser distribuidos |
| Vendor management | Preferir vendors existentes |
| Data residency | Datos de cliente en-country |

### Sindicato
- Personal de tiendas sindicalizado
- Cambios en herramientas requieren negociacion
- Capacitacion en horario laboral
```
