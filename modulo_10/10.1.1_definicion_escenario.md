# Definicion del Escenario de Proyecto

**Tiempo estimado**: 45 minutos
**Nivel**: Avanzado
**Prerrequisitos**: Todos los modulos anteriores (1-9)

---

## Por que importa este subtema?

El proyecto integrador es donde se aplican todos los conocimientos adquiridos en un escenario realista. Definir correctamente el escenario es crucial para practicar el ciclo ADM completo en un contexto que simule los desafios reales de arquitectura empresarial.

---

## Objetivo del Proyecto Integrador

> "Aplicar el ciclo ADM completo para desarrollar una arquitectura empresarial que responda a un escenario de transformacion de negocio."

**Entregables**:
1. Architecture Vision Document
2. Architecture Definition Document (4 dominios)
3. Gap Analysis
4. Architecture Roadmap
5. Presentacion ejecutiva

---

## Escenario: Transformacion Digital de RetailCorp

### Contexto de la Organizacion

```markdown
## RetailCorp S.A.

### Perfil
| Aspecto | Descripcion |
|---------|-------------|
| Industria | Retail (tiendas departamentales) |
| Ingresos | $2.5 billones anuales |
| Empleados | 15,000 |
| Tiendas | 85 tiendas fisicas |
| Presencia | Nacional (3 regiones) |
| Ecommerce | 8% de ventas (creciendo) |

### Historia
- Fundada en 1975 como tienda familiar
- Expansion agresiva 1990-2010
- Lider en mercado tradicional
- Ultimos 5 anos: perdida de market share vs nativos digitales

### Situacion Actual
- Sistemas legacy de 15+ anos
- ERP monolitico (SAP ECC 6.0)
- POS propietario en tiendas
- Ecommerce separado (Magento on-premise)
- Experiencia cliente fragmentada
- Time-to-market lento (6+ meses para nuevas features)
```

### Drivers de Transformacion

```markdown
## Por que transformar ahora

### Presiones de Negocio
| Driver | Impacto | Urgencia |
|--------|---------|----------|
| Competencia digital | Perdiendo 2% market share/ano | Alta |
| Expectativas cliente | NPS cayendo 10 puntos | Alta |
| Costos de TI | 30% del budget en mantenimiento legacy | Media |
| Agilidad | Competidores lanzan features en semanas | Alta |

### Oportunidades
| Oportunidad | Potencial |
|-------------|-----------|
| Omnicanalidad | +15% en ventas cross-canal |
| Personalizacion | +8% conversion rate |
| Supply chain | -12% costos de inventario |
| Data analytics | Nuevos insights de cliente |

### Mandato Estrategico
"Convertirnos en un retailer digital-first para 2028,
manteniendo la excelencia en experiencia de tienda fisica."
- CEO, Plan Estrategico 2026-2030
```

### Estado Actual (Baseline Architecture)

```markdown
## Arquitectura Actual

### Business Architecture
- 12 procesos core de retail
- Operaciones de tienda manuales
- Supply chain reactivo
- Marketing tradicional (sin personalizacion)
- Servicio al cliente en silos

### Application Architecture
| Sistema | Tecnologia | Edad | Estado |
|---------|------------|------|--------|
| ERP (SAP) | ABAP | 18 anos | EOL en 2027 |
| POS | C++ propietario | 12 anos | Inestable |
| Ecommerce | Magento 1.9 | 8 anos | Sin soporte |
| CRM | Salesforce Classic | 10 anos | Desactualizado |
| WMS | Oracle WMS | 15 anos | Funcional |
| BI | Cognos | 10 anos | Limitado |

### Data Architecture
- Datos en silos por sistema
- Sin MDM (multiples maestros de producto/cliente)
- Batch processing nocturno
- No hay data lake
- Reportes manuales en Excel

### Technology Architecture
- Datacenter on-premise (2 ubicaciones)
- Redes WAN tradicionales a tiendas
- Servidores fisicos (virtualizacion parcial)
- No hay cloud
- Backup en cinta
```

### Vision Futura (Target State)

```markdown
## Arquitectura Objetivo (2028)

### Business Architecture
- Experiencia omnicanal unificada
- Supply chain predictivo
- Marketing personalizado real-time
- Operaciones de tienda digitalizadas
- Customer 360

### Application Architecture
| Capacidad | Solucion Target |
|-----------|-----------------|
| Core Commerce | Comercetools (headless) |
| ERP | SAP S/4HANA Cloud |
| POS | Cloud POS integrado |
| CRM | Salesforce CDP |
| WMS | Manhattan Active |
| Analytics | Snowflake + Tableau |

### Data Architecture
- Customer Data Platform centralizado
- Product Information Management (PIM)
- Event streaming (real-time)
- Data Lake para analytics
- ML/AI para personalizacion

### Technology Architecture
- Cloud-first (AWS primary)
- Kubernetes para workloads
- API Gateway centralizado
- Edge computing en tiendas
- Zero-trust security
```

## Scope del Proyecto

### Alcance

```markdown
## Que INCLUYE el proyecto

### Dominios
- [x] Business Architecture (foco en customer journey)
- [x] Data Architecture (foco en Customer 360)
- [x] Application Architecture (portafolio completo)
- [x] Technology Architecture (cloud migration)

### Fases ADM
- [x] Preliminary
- [x] Phase A: Architecture Vision
- [x] Phase B: Business Architecture
- [x] Phase C: IS Architecture (Data + Application)
- [x] Phase D: Technology Architecture
- [x] Phase E: Opportunities and Solutions
- [x] Phase F: Migration Planning
- [ ] Phase G: Implementation Governance (mencion)
- [ ] Phase H: Change Management (mencion)

### Entregables
- Architecture Vision Document
- Business Architecture (capacidades, procesos clave)
- Data Architecture (modelo conceptual, flujos)
- Application Architecture (portafolio, integraciones)
- Technology Architecture (infraestructura target)
- Gap Analysis consolidado
- Architecture Roadmap (3 anos)
```

### Fuera de Alcance

```markdown
## Que NO INCLUYE el proyecto

- Diseno detallado de sistemas individuales
- Seleccion de vendors (solo recomendaciones)
- Implementacion
- Presupuesto detallado
- Gestion del cambio organizacional
- Governance post-implementacion
```
