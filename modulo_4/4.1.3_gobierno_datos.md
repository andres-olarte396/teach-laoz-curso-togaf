# Gobierno de Datos

**Tiempo estimado**: 25 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Modelado Logico de Datos (4.1.2)

---

## Por que importa este subtema?

Modelar datos es solo el principio. Sin gobierno, los datos se degradan rapidamente:
- Duplicados y inconsistencias
- Datos obsoletos o incorrectos
- Uso no autorizado
- Incumplimiento regulatorio

El gobierno de datos establece **quien**, **que**, **como** y **cuando** se gestionan los datos.

---

## Que es Gobierno de Datos

> "El ejercicio de autoridad, control y toma de decisiones compartida sobre la gestion de activos de datos."

**Componentes**:
- **Politicas**: Reglas que rigen el uso de datos
- **Estandares**: Especificaciones tecnicas y de calidad
- **Procesos**: Como se ejecutan las actividades
- **Roles**: Quien es responsable de que
- **Metricas**: Como se mide el cumplimiento

## Roles de Gobierno

### Data Owner (Propietario de Datos)

| Aspecto | Descripcion |
|---------|-------------|
| Quien | Lider de negocio del dominio |
| Responsabilidad | Definir reglas, aprobar acceso |
| Autoridad | Decisiones sobre uso y calidad |
| Ejemplo | VP de Ventas es owner de datos de Cliente |

### Data Steward (Custodio de Datos)

| Aspecto | Descripcion |
|---------|-------------|
| Quien | Experto de dominio operativo |
| Responsabilidad | Implementar politicas, monitorear calidad |
| Autoridad | Operativa, escala a Owner |
| Ejemplo | Analista de CRM es steward de datos de Cliente |

### Data Custodian (Administrador Tecnico)

| Aspecto | Descripcion |
|---------|-------------|
| Quien | TI / DBA |
| Responsabilidad | Implementar controles tecnicos |
| Autoridad | Tecnica |
| Ejemplo | DBA administra base de datos de clientes |

## Politicas Clave

### Politica de Calidad de Datos

```markdown
## Politica: Calidad de Datos de Cliente

### Alcance
Aplica a todos los datos del dominio Cliente en todos los sistemas.

### Dimensiones de Calidad

| Dimension | Definicion | Umbral |
|-----------|------------|--------|
| Completitud | Campos obligatorios llenos | >98% |
| Exactitud | Datos correctos vs realidad | >95% |
| Consistencia | Mismo valor en todos los sistemas | >99% |
| Unicidad | Sin duplicados | >99.5% |
| Oportunidad | Datos actualizados en <24h | >95% |

### Responsabilidades
- Data Owner: Aprobar umbrales
- Data Steward: Monitorear y remediar
- Usuarios: Reportar problemas

### Proceso de Remediacion
1. Detectar via reporte automatico
2. Asignar a Steward
3. Investigar causa raiz
4. Corregir datos
5. Implementar control preventivo
```

### Politica de Clasificacion de Datos

```markdown
## Politica: Clasificacion de Datos

### Niveles de Clasificacion

| Nivel | Descripcion | Controles |
|-------|-------------|-----------|
| PUBLICO | Informacion publica | Ninguno especial |
| INTERNO | Solo empleados | Autenticacion |
| CONFIDENCIAL | Necesidad de saber | Autenticacion + autorizacion |
| RESTRINGIDO | Altamente sensible | Encriptacion + MFA + auditoria |

### Clasificacion por Tipo de Dato

| Tipo de Dato | Clasificacion |
|--------------|---------------|
| Precios publicados | PUBLICO |
| Ventas internas | INTERNO |
| Datos de cliente (PII) | CONFIDENCIAL |
| Datos de pago (PCI) | RESTRINGIDO |
| Datos de salud (PHI) | RESTRINGIDO |

### Requisitos por Clasificacion

| Aspecto | CONFIDENCIAL | RESTRINGIDO |
|---------|--------------|-------------|
| Encriptacion en reposo | Recomendado | Obligatorio |
| Encriptacion en transito | Obligatorio | Obligatorio |
| Logs de acceso | Recomendado | Obligatorio |
| Retencion | Segun politica | Minimo legal |
| Transferencia externa | Aprobacion Owner | Prohibida sin contrato |
```
