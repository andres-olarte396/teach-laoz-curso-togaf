# Modelado de Procesos de Negocio

**Tiempo estimado**: 35 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Modelado de Capacidades de Negocio (3.1.1)

---

## Por que importa este subtema?

Mientras las capacidades describen "que" puede hacer el negocio, los procesos describen "como" lo hace. En la Fase B del ADM, modelar procesos es esencial para:

1. Entender el estado actual (baseline)
2. Disenar mejoras (target)
3. Identificar requerimientos para sistemas
4. Validar que la arquitectura soporta las operaciones

---

## Que es un Proceso de Negocio

> "Un conjunto de actividades interrelacionadas que transforman inputs en outputs para entregar valor a un cliente o stakeholder."

**Caracteristicas**:
- Tiene un **trigger** (evento que lo inicia)
- Tiene **inputs** y **outputs** definidos
- Involucra **roles** especificos
- Puede medirse en **tiempo, costo, calidad**

## Niveles de Detalle

```
NIVEL 0: Value Stream (Cadena de Valor)
    |
    +-- NIVEL 1: Proceso End-to-End
            |
            +-- NIVEL 2: Subproceso
                    |
                    +-- NIVEL 3: Actividad
                            |
                            +-- NIVEL 4: Tarea (para procedimientos)
```

### Ejemplo: Proceso de Pedidos

```
N0: Cadena de Valor (Pedido-a-Cobro)
    |
    +-- N1: Proceso de Gestion de Pedidos
            |
            +-- N2: Captura de Pedido
            |       +-- N3: Validar cliente
            |       +-- N3: Validar disponibilidad
            |       +-- N3: Calcular precio
            |
            +-- N2: Fulfillment
            |       +-- N3: Reservar inventario
            |       +-- N3: Generar orden de picking
            |       +-- N3: Preparar envio
            |
            +-- N2: Facturacion
                    +-- N3: Generar factura
                    +-- N3: Enviar al cliente
                    +-- N3: Registrar en contabilidad
```

## Notacion BPMN

TOGAF no prescribe una notacion, pero BPMN (Business Process Model and Notation) es el estandar mas usado.

### Elementos basicos

```
EVENTOS (Circulos)
  (O)  Evento de inicio (verde)
  (O)  Evento intermedio (amarillo)
  (X)  Evento de fin (rojo)

ACTIVIDADES (Rectangulos redondeados)
  +-------+
  | Tarea |
  +-------+

  +-------+
  |[+]    |  Subproceso (con +)
  +-------+

GATEWAYS (Rombos)
  < >  Exclusivo (XOR) - solo un camino
  <+>  Paralelo (AND) - todos los caminos
  <o>  Inclusivo (OR) - uno o mas caminos

FLUJOS (Flechas)
  ---->  Flujo de secuencia
  - - -> Flujo de mensaje
  .....> Asociacion
```

### Ejemplo BPMN simple

```
(O)---> [Recibir  ] ---> < > ---> [Aprobar    ] ---> (X)
Inicio   Solicitud       |         Manual
                         |
                         +------> [Aprobar    ] ---> (X)
                                   Automatico
```

## Proceso Baseline vs Target

### Baseline (AS-IS)

Documenta como opera el proceso **hoy**, incluyendo:
- Pasos manuales y workarounds
- Sistemas legacy
- Ineficiencias conocidas
- Tiempos reales (no ideales)

### Target (TO-BE)

Documenta como **deberia** operar el proceso:
- Automatizaciones propuestas
- Nuevos sistemas
- Eliminacion de pasos innecesarios
- Mejoras de tiempo/costo/calidad

### Ejemplo comparativo

```
BASELINE (AS-IS):
(O)-> [Recibir  ] -> [Ingresar ] -> [Verificar] -> [Aprobar ] -> (X)
      solicitud     a Excel       manual       por email
      (email)       (5 min)       (30 min)     (2 dias)

      Tiempo total: 2.5 dias
      Sistemas: Email, Excel
      Errores: 15%

TARGET (TO-BE):
(O)-> [Capturar    ] -> < > -> [Auto-aprobar] -> (X)
      en portal         |       (instantaneo)
      (2 min)           |
                        +----> [Workflow    ] -> [Aprobar] -> (X)
                               aprobacion       (4 horas)
                               (excepcion)

      Tiempo total: 2 min - 4 horas
      Sistemas: Portal, Motor reglas, Workflow
      Errores: <1%
```

## Errores Comunes

| Error | Consecuencia | Prevencion |
|-------|--------------|------------|
| Modelar solo el "happy path" | Excepciones no manejadas | Incluir flujos alternativos |
| Demasiado detalle en N1 | Diagrama ilegible | Usar subprocesos |
| No involucrar a ejecutores | Proceso irreal | Validar con quienes ejecutan |
| Baseline idealizado | Gaps incorrectos | Documentar realidad, no ideal |
| Sin metricas | No se puede medir mejora | Definir KPIs desde el inicio |
