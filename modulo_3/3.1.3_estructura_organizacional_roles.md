# Estructura Organizacional y Roles

**Tiempo estimado**: 25 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Modelado de Procesos de Negocio (3.1.2)

---

## Por que importa este subtema?

La arquitectura de negocio no solo incluye capacidades y procesos, sino tambien **quien** los ejecuta. La estructura organizacional define:

- Que unidades existen y como se relacionan
- Que roles ejecutan que actividades
- Donde estan las responsabilidades y autoridades

Sin este modelado, la arquitectura queda incompleta y no se puede evaluar el impacto del cambio en las personas.

---

## Conceptos Clave

### Unidad Organizacional

> "Un grupo autocontenido de personas organizadas para un proposito particular."

Ejemplos: Departamento, Division, Equipo, Centro de Competencia

### Actor

> "Una persona, organizacion o sistema que tiene un comportamiento."

Ejemplos: Cliente, Proveedor, Empleado, Sistema externo

### Rol

> "La responsabilidad usual o funcion de un actor en una interaccion particular."

Ejemplos: Gerente de Ventas, Analista de Credito, Aprobador

---

## Diferencia entre Actor y Rol

```
ACTOR: Juan Perez (persona especifica)
       |
       +-- ROL 1: Gerente de Ventas (en proceso de ventas)
       +-- ROL 2: Aprobador de Descuentos (en proceso de pricing)
       +-- ROL 3: Owner de P&L Regional (en proceso de planeacion)

Un actor puede tener multiples roles.
Un rol puede ser asumido por multiples actores.
```

### Organigrama tradicional

```
                    +-------------+
                    |    CEO      |
                    +------+------+
                           |
          +----------------+----------------+
          |                |                |
    +-----+-----+    +-----+-----+    +-----+-----+
    | VP Ventas |    | VP Ops    |    | VP Tech   |
    +-----+-----+    +-----+-----+    +-----+-----+
          |                |                |
    +-----+-----+    +-----+-----+    +-----+-----+
    | Dir Reg   |    | Dir Prod  |    | Dir Dev   |
    | Norte     |    |           |    |           |
    +-----------+    +-----------+    +-----------+
```

### Matriz de responsabilidades (RACI)

| Proceso/Actividad | Ventas | Operaciones | Finanzas | TI |
|-------------------|--------|-------------|----------|-----|
| Capturar pedido | R | I | - | S |
| Validar credito | C | - | R | S |
| Fulfillment | I | R | - | S |
| Facturar | I | A | R | S |
| Soporte sistema | - | C | - | R |

**R** = Responsable (ejecuta)
**A** = Accountable (autoridad final)
**C** = Consultado (opina)
**I** = Informado (recibe info)
**S** = Soporte (ayuda)

## Impacto Organizacional en Arquitectura

Al disenar el target, considerar:

### Nuevos roles que se crean

```
BASELINE:
- Ingreso manual de pedidos (Capturista)

TARGET:
- Pedidos automaticos via portal
- NUEVO ROL: Administrador de Excepciones
- ELIMINADO: Capturista (reasignar o reducir)
```

### Cambios en responsabilidades

```
BASELINE:
- Aprobacion de credito: Solo Analista de Credito

TARGET:
- Auto-aprobacion para montos < $10K
- Aprobacion humana solo para excepciones
- CAMBIO: Analista de Credito ahora maneja solo excepciones complejas
```

### Requerimientos de capacitacion

| Rol | Cambio | Capacitacion requerida |
|-----|--------|------------------------|
| Ejecutivo de Cuenta | Nuevo portal | 4 horas uso del portal |
| Analista de Credito | Nuevas reglas | 8 horas politicas de credito |
| Aprobador | Nuevo workflow | 2 horas aprobacion digital |
