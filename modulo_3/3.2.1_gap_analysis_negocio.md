# Gap Analysis de Negocio

**Tiempo estimado**: 30 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Estructura Organizacional y Roles (3.1.3)

---

## Por que importa este subtema?

El Gap Analysis es la tecnica que **transforma** la arquitectura de un ejercicio teorico a un plan de accion. Al comparar sistematicamente el baseline con el target, identificamos exactamente:

- Que falta construir (gaps)
- Que sobra eliminar (redundancias)
- Que debe modificarse (transformaciones)

Sin gap analysis, no hay roadmap concreto.

---

## Que es Gap Analysis

> "La tecnica de comparar dos estados de arquitectura (baseline y target) para identificar las diferencias que deben abordarse."

**Formula conceptual**:
```
GAPS = TARGET - BASELINE

Donde cada gap es:
- NUEVO: Existe en Target, no en Baseline
- ELIMINAR: Existe en Baseline, no en Target
- MODIFICAR: Existe en ambos pero diferente
- MANTENER: Existe en ambos, sin cambios
```

---

## Proceso de Gap Analysis

```
1. PREPARAR
   - Tener Baseline documentado
   - Tener Target documentado
   - Definir criterios de comparacion
         |
         v
2. COMPARAR
   - Elemento por elemento
   - Identificar diferencias
   - Clasificar tipo de gap
         |
         v
3. ANALIZAR
   - Evaluar impacto de cada gap
   - Identificar dependencias
   - Priorizar
         |
         v
4. DOCUMENTAR
   - Gap Analysis Matrix
   - Narrativa de hallazgos
   - Recomendaciones
```

## Gap Analysis Matrix

### Formato basico

```
                         TARGET STATE
                   +------------------------+
                   | Existe    | No Existe  |
         +---------+-----------+------------+
BASELINE | Existe  | MANTENER/ | ELIMINAR   |
STATE    |         | MODIFICAR |            |
         +---------+-----------+------------+
         | No      | NUEVO     | N/A        |
         | Existe  |           |            |
         +---------+-----------+------------+
```

### Ejemplo: Capacidades de Negocio

| Capacidad | Baseline | Target | Gap Type | Accion |
|-----------|----------|--------|----------|--------|
| Gestion de Clientes | Existe (CRM Siebel) | Existe (Salesforce) | MODIFICAR | Migrar a Salesforce |
| Gestion de Leads | No existe | Existe | NUEVO | Implementar en Salesforce |
| Facturacion Manual | Existe | No existe | ELIMINAR | Retirar proceso manual |
| Gestion de Pagos | Existe | Existe (sin cambios) | MANTENER | Sin accion |

### Ejemplo: Procesos

| Proceso | Baseline | Target | Gap | Impacto |
|---------|----------|--------|-----|---------|
| Cotizacion | Manual, 5 dias | Automatico, 4h | Automatizar | Alto |
| Aprobacion credito | Manual | Reglas + excepciones | Parcialmente auto | Medio |
| Facturacion | Semi-auto | Totalmente auto | Eliminar pasos | Medio |
| Cobranza | Existe | Sin cambios | Mantener | Bajo |

### Ejemplo: Roles

| Rol | Baseline | Target | Gap | Impacto RRHH |
|-----|----------|--------|-----|--------------|
| Capturista de pedidos | 10 FTE | 0 FTE | ELIMINAR | Reasignar 10 personas |
| Analista de excepciones | No existe | 3 FTE | NUEVO | Contratar o reasignar |
| Ejecutivo de ventas | 20 FTE | 20 FTE | MODIFICAR | Capacitar en nuevo sistema |

## Detalle de Gap

Para cada gap significativo, documentar:

```markdown
## GAP-BUS-001: Automatizacion de Cotizaciones

### Descripcion
El proceso actual de cotizaciones es manual y toma 5 dias.
El target requiere un proceso automatizado que tome menos de 4 horas.

### Estado Baseline
- Proceso: Manual con Excel y emails
- Tiempo: 5 dias promedio
- Errores: 8%
- Sistemas: Excel, Email, ERP (consulta manual)

### Estado Target
- Proceso: Automatizado con portal y reglas
- Tiempo: <4 horas (80% automatico)
- Errores: <1%
- Sistemas: Portal web, Motor de reglas, ERP (integracion API)

### Diferencias Clave
1. Eliminacion de ingreso manual de datos
2. Integracion automatica con precios del ERP
3. Workflow de aprobacion electronico
4. Portal de seguimiento para clientes

### Impacto
- **Negocio**: Aumento conversion 15%
- **Operaciones**: Reduccion 80% tiempo de proceso
- **RRHH**: 5 FTE reasignados de captura a analisis
- **TI**: Desarrollo de portal + integraciones

### Dependencias
- GAP-APP-003: API de precios del ERP
- GAP-DATA-001: Master de productos unificado
- GAP-TECH-002: Infraestructura cloud

### Esfuerzo Estimado
- Desarrollo: 3 meses
- Capacitacion: 2 semanas
- Piloto: 1 mes

### Prioridad: ALTA
### Riesgo: MEDIO
```

## Tecnicas de Priorizacion

### Matriz Impacto-Esfuerzo

```
              ESFUERZO
              Bajo         Alto
        +------------+------------+
   Alto | QUICK WINS | PROYECTOS  |
IMPACTO |   (Hacer   |  MAYORES   |
        |   primero) |            |
        +------------+------------+
   Bajo | FILL-INS   | EVITAR o   |
        | (Si hay    |  DIFERIR   |
        |  tiempo)   |            |
        +------------+------------+
```

### Criterios de priorizacion

| Criterio | Peso | GAP-001 | GAP-002 | GAP-003 |
|----------|------|---------|---------|---------|
| Valor de negocio | 30% | 9 | 5 | 7 |
| Urgencia | 25% | 8 | 3 | 6 |
| Reduccion de riesgo | 20% | 7 | 8 | 4 |
| Facilidad de implementacion | 15% | 6 | 9 | 5 |
| Dependencias | 10% | 5 | 7 | 8 |
| **Score ponderado** | 100% | **7.4** | **5.8** | **6.0** |
| **Prioridad** | | 1 | 3 | 2 |

## Errores Comunes

| Error | Consecuencia | Prevencion |
|-------|--------------|------------|
| Baseline incompleto | Gaps ocultos | Validar baseline con operaciones |
| Target sin detallar | Gaps vagos | Especificar target al mismo nivel que baseline |
| Ignorar dependencias | Secuencia incorrecta | Mapear dependencias entre gaps |
| No involucrar a negocio | Gaps tecnicos sin contexto | Workshops con stakeholders |
| Demasiados gaps | Paralisis | Agrupar en work packages |
