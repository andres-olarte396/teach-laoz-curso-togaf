# Skills y Desarrollo de Arquitectos

**Tiempo estimado**: 25 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Roles del Equipo de Arquitectura (9.1.1)

---

## Por que importa este subtema?

La efectividad de la practica de arquitectura depende directamente de las competencias del equipo. Un programa estructurado de desarrollo de skills asegura que los arquitectos tengan las capacidades necesarias para cumplir su rol.

## Meta Skills

### Systems Thinking

```markdown
## Systems Thinking

### Componentes
- Ver el todo, no solo las partes
- Entender feedback loops
- Identificar efectos de segundo orden
- Reconocer patrones emergentes
- Modelar complejidad

### Ejercicios
| Ejercicio | Proposito |
|-----------|-----------|
| Causal loop diagrams | Visualizar feedback |
| Impact mapping | Efectos en cascada |
| Wardley mapping | Evolucion de componentes |
| Rich pictures | Capturar complejidad |
```

### Problem Solving

```markdown
## Problem Solving Framework

### Proceso
1. **Define**: Cual es realmente el problema?
2. **Explore**: Cuales son las opciones?
3. **Evaluate**: Trade-offs de cada opcion
4. **Decide**: Seleccionar con criterios claros
5. **Validate**: Verificar que se resolvio

### Tecnicas
| Tecnica | Uso |
|---------|-----|
| 5 Whys | Root cause analysis |
| MECE | Estructurar problemas |
| Trade-off analysis | Comparar opciones |
| Prototype/POC | Validar soluciones |
| Post-mortem | Aprender de errores |
```


### Criterios por Nivel

```markdown
## Criterios de Promocion

### Junior -> Mid
- [ ] 2+ anos de experiencia
- [ ] Entregado 3+ proyectos con supervision
- [ ] Certificacion TOGAF Foundation
- [ ] Domina 1 dominio tecnico
- [ ] Documenta efectivamente

### Mid -> Senior
- [ ] 5+ anos de experiencia
- [ ] Liderado 5+ proyectos independientemente
- [ ] Certificacion TOGAF Certified
- [ ] Domina 2+ dominios
- [ ] Mentoring de juniors
- [ ] Contribuye a estandares

### Senior -> Principal
- [ ] 8+ anos de experiencia
- [ ] Impacto a nivel enterprise
- [ ] Define estandares y estrategia
- [ ] Desarrolla otros arquitectos
- [ ] Reconocido fuera del equipo
- [ ] Presenta en Architecture Board
```
