# Triggers de Cambio en Arquitectura

**Tiempo estimado**: 25 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Compliance Reviews (7.1.2)

---

## Por que importa este subtema?

La arquitectura no es estatica. Eventos internos y externos disparan la necesidad de cambios. Identificar estos triggers permite responder proactivamente en lugar de reactivamente, manteniendo la arquitectura alineada con las necesidades del negocio.

---

## Que son los Triggers de Cambio

> "Eventos, condiciones o circunstancias que inician la necesidad de revisar, actualizar o transformar la arquitectura empresarial."

**Importancia**:
- Anticipar necesidades de cambio
- Planificar recursos adecuadamente
- Evitar arquitectura obsoleta
- Mantener relevancia de negocio

## Triggers Externos

### 1. Triggers de Mercado

```markdown
## Triggers de Mercado

| Trigger | Ejemplo | Impacto Arquitectura |
|---------|---------|----------------------|
| Nuevo competidor | Fintech entra al mercado | Acelerar transformacion digital |
| Cambio en demanda | Clientes quieren self-service | Desarrollar portal cliente |
| Fusion/adquisicion competidor | Competidor compra proveedor | Diversificar proveedores |
| Nuevo canal | Adopcion masiva de app | Mobile-first architecture |
| Globalizacion | Expansion a LATAM | Multi-region, multi-currency |

### Caso: Entrada de Competidor Digital

**Situacion**: Neobank lanza producto similar con mejor UX
**Trigger**: Perdida de market share proyectada
**Respuesta arquitectura**:
- Acelerar modernizacion de front-end
- Implementar API banking
- Reducir time-to-market
```

### 2. Triggers Regulatorios

```markdown
## Triggers Regulatorios

| Regulacion | Region | Impacto Arquitectura |
|------------|--------|----------------------|
| GDPR | EU | Data governance, consent management |
| CCPA | California | Derechos de datos, borrado |
| PSD2/Open Banking | EU | APIs abiertas, seguridad |
| SOX | USA | Controles financieros, auditoria |
| DORA | EU | Resiliencia operativa digital |

### Caso: Cumplimiento GDPR

**Situacion**: Nueva regulacion de privacidad en Europa
**Trigger**: Fecha limite de compliance
**Respuesta arquitectura**:
- Inventario de datos personales
- Implementar consent management
- Capacidad de borrado (right to be forgotten)
- Data lineage y auditoria
- Privacy by design en nuevos sistemas
```

### 3. Triggers Tecnologicos

```markdown
## Triggers Tecnologicos

| Trigger | Ejemplo | Impacto |
|---------|---------|---------|
| Nueva tecnologia | AI generativa | Explorar casos de uso |
| Obsolescencia | Windows Server 2012 EOL | Migrar a versiones soportadas |
| Vulnerabilidad critica | Log4Shell | Patch urgente, revision de dependencias |
| Cambio de proveedor | Oracle -> PostgreSQL | Migracion de base de datos |
| Cloud maturity | Nuevos servicios AWS | Evaluar adopcion |

### Caso: End of Support

**Situacion**: SAP ECC end of support 2027
**Trigger**: Fecha limite de soporte del vendor
**Respuesta arquitectura**:
- Evaluar S/4HANA vs alternativas
- Plan de migracion multi-anual
- Modernizar integraciones
- Revisar customizaciones
```

## Triggers Internos

### 1. Triggers Estrategicos

```markdown
## Triggers Estrategicos

| Trigger | Ejemplo | Impacto |
|---------|---------|---------|
| Nueva estrategia | "Digital first" | Transformacion de arquitectura |
| M&A | Adquisicion de startup | Integracion de sistemas |
| Reorganizacion | Nueva unidad de negocio | Reestructurar dominios |
| Nuevo producto | Lanzamiento linea seguros | Nueva capability |
| Desinversion | Venta de division | Separacion de sistemas |

### Caso: Adquisicion

**Situacion**: Empresa adquiere startup fintech
**Trigger**: Cierre de la adquisicion
**Respuesta arquitectura**:
- Assessment de arquitectura adquirida
- Identificar sinergias y redundancias
- Plan de integracion (o coexistencia)
- Unificar plataformas comunes
- Decidir que sistemas mantener
```

### 2. Triggers Operacionales

```markdown
## Triggers Operacionales

| Trigger | Indicador | Umbral | Accion |
|---------|-----------|--------|--------|
| Performance | Latencia p95 | > 500ms | Optimizacion/scaling |
| Disponibilidad | Uptime | < 99.9% | HA improvements |
| Incidentes | Severidad 1 por mes | > 2 | Root cause architecture |
| Capacidad | Utilizacion CPU | > 80% | Scaling horizontal |
| Deuda tecnica | Indice sonar | > 20 dias | Refactoring sprint |

### Caso: Incidente Critico Recurrente

**Situacion**: Tercer incidente de perdida de datos en 6 meses
**Trigger**: Pattern de incidentes
**Respuesta arquitectura**:
- Root cause analysis
- Revisar arquitectura de persistencia
- Implementar event sourcing
- Mejorar disaster recovery
```

### 3. Triggers Financieros

```markdown
## Triggers Financieros

| Trigger | Situacion | Respuesta |
|---------|-----------|-----------|
| Reduccion presupuesto | Recorte 20% IT | Racionalizar portafolio |
| Costo de licencias | Oracle sube 40% | Evaluar alternativas |
| ROI negativo | Proyecto no rentable | Decommission |
| Oportunidad ahorro | Cloud vs on-prem | Business case migracion |

### Caso: Optimizacion de Costos

**Situacion**: Presion para reducir costos IT 30%
**Trigger**: Objetivo financiero corporativo
**Respuesta arquitectura**:
- Racionalizar portafolio de aplicaciones
- Consolidar infraestructura
- Migrar a cloud (pay per use)
- Automatizar operaciones
- Renegociar contratos
```

### 4. Triggers Tecnicos

```markdown
## Triggers Tecnicos

| Trigger | Indicador | Accion |
|---------|-----------|--------|
| Deuda tecnica critica | > 30 dias en SonarQube | Sprint de refactoring |
| Dependencias obsoletas | Librerias sin soporte | Actualizacion |
| Complejidad | Cyclomatic > 50 | Rediseno |
| Acoplamiento | Dependencias circulares | Desacoplar |
| Conocimiento | Single point of failure | Documentar, capacitar |

### Caso: Deuda Tecnica Acumulada

**Situacion**: Sistema core con 15 anos de parches
**Trigger**: Imposible implementar nuevos requisitos
**Respuesta arquitectura**:
- Strangler fig pattern para modernizar
- Microservicios incrementales
- Plan multi-anual de reemplazo
```

## Matriz de Priorizacion de Triggers

```
                    URGENCIA
                    Alta          Baja
              +------------+------------+
        Alta  |  CRITICO   |  ESTRATEGICO|
IMPACTO       |  (Actuar   |  (Planificar|
              |   ahora)   |   bien)     |
              +------------+------------+
        Baja  |  TACTICO   |  MONITOREAR |
              |  (Resolver |  (Backlog)  |
              |   rapido)  |             |
              +------------+------------+

CRITICO: Regulatorio con deadline, incidente critico
ESTRATEGICO: M&A, nueva estrategia
TACTICO: Optimizacion de costos, upgrade tecnico
MONITOREAR: Tendencias tecnologicas
```
