# Establecimiento del Framework

**Tiempo estimado**: 20 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Principios de Arquitectura (2.2.2)

---

## Por que importa este subtema?

TOGAF es un framework generico. Antes de usarlo, cada organizacion debe **adaptarlo y configurarlo** para su contexto. Este proceso de "tailoring" produce el **Tailored Architecture Framework** que sera la base de todos los proyectos de arquitectura.

---

## Que se Configura

### 1. Proceso (ADM)

| Elemento | Configuracion |
|----------|---------------|
| Fases | Cuales usar, cuales omitir |
| Steps | Cuales son obligatorios, cuales opcionales |
| Iteraciones | Como manejar ciclos |
| Governance | Puntos de revision y aprobacion |

### 2. Contenido (Content Framework)

| Elemento | Configuracion |
|----------|---------------|
| Entregables | Cuales son obligatorios |
| Artefactos | Formato y nivel de detalle |
| Viewpoints | Cuales usar |
| Notacion | ArchiMate, UML, otra |

### 3. Repositorio

| Elemento | Configuracion |
|----------|---------------|
| Herramienta | Confluence, SharePoint, EA tool |
| Estructura | Organizacion de carpetas |
| Clasificacion | Como categorizar contenido |
| Acceso | Quien puede ver/editar que |

### 4. Terminologia

| TOGAF | Adaptado (ejemplo) |
|-------|-------------------|
| Architecture Building Block | Componente de Arquitectura |
| Statement of Architecture Work | Acta de Inicio |
| Architecture Board | Comite de Arquitectura |
| Concern | Requisito/Preocupacion |

## Ejemplo: ADM por Tipo de Proyecto

```markdown
## Tipos de Proyecto y ADM Aplicable

### Tipo 1: Estrategico (Enterprise-wide)
- **Duracion**: 12-24 meses
- **Fases**: Todas
- **Governance**: Reviews mensuales, aprobacion por Board
- **Entregables**: Completos

### Tipo 2: Segmento (Division/Unidad de Negocio)
- **Duracion**: 4-8 meses
- **Fases**: A, B, C, D (paralelo), E, F, G
- **Governance**: Reviews quincenales, aprobacion por Chief Architect
- **Entregables**: Simplificados

### Tipo 3: Capacidad (Sistema/Funcionalidad)
- **Duracion**: 4-12 semanas
- **Fases**: A (reducida), C o D, E+F (combinadas)
- **Governance**: Review al inicio y fin
- **Entregables**: ADRs + Diagrams

### Tipo 4: Emergency/Tactical
- **Duracion**: 1-2 semanas
- **Fases**: A (1 dia), Decision, Documentacion retroactiva
- **Governance**: Aprobacion verbal, formalizacion posterior
- **Entregables**: ADR minimo
```

## Proceso de Establecimiento

```
1. ANALIZAR
   - Evaluar madurez actual
   - Identificar practicas existentes
   - Entrevistar stakeholders

2. DISENAR
   - Definir modelo organizacional
   - Adaptar ADM
   - Seleccionar herramientas
   - Crear templates

3. DOCUMENTAR
   - Escribir Tailored Framework
   - Crear guias de uso
   - Desarrollar materiales de capacitacion

4. PILOTEAR
   - Aplicar en proyecto piloto
   - Recoger feedback
   - Ajustar

5. DESPLEGAR
   - Comunicar a la organizacion
   - Capacitar equipos
   - Establecer soporte

6. EVOLUCIONAR
   - Revisar periodicamente
   - Incorporar lecciones aprendidas
   - Actualizar segun necesidades
```
