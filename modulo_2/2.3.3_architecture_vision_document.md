# Architecture Vision Document

**Tiempo estimado**: 25 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Stakeholder Management (2.3.2)

---

## Por que importa este subtema?

El Architecture Vision es el **entregable mas importante de la Fase A**. Es el documento que:

1. Comunica el "que" y el "por que" de la arquitectura
2. Obtiene buy-in de stakeholders clave
3. Establece la linea base para medir el exito
4. Autoriza el trabajo de arquitectura detallado

Sin una vision clara y aprobada, el proyecto carece de direccion.

---

## Que es el Architecture Vision

> "Una descripcion de alto nivel de las arquitecturas baseline y target, destacando las diferencias clave y el valor de negocio que se entregara."

**Caracteristicas**:
- Alto nivel (no detalle tecnico)
- Orientado a negocio
- Comprensible por ejecutivos
- Motivador y convincente

## Ejemplo Completo

```markdown
# Architecture Vision
# Sistema de Cotizaciones Automatizado

## 1. Executive Summary

Este documento presenta la vision de arquitectura para transformar
el proceso de cotizaciones de [Empresa], reduciendo el tiempo de
respuesta de 5 dias a 24 horas, y eliminando errores manuales de
pricing que cuestan $500K anuales.

La solucion propuesta integra los sistemas existentes (Salesforce,
SAP) con un nuevo motor de pricing y workflow automatizado, manteniendo
las inversiones actuales mientras agrega capacidades criticas.

**Inversion requerida**: $430K
**ROI esperado**: 18 meses
**Go-live objetivo**: Q4 2026

---

## 2. Contexto y Drivers

### 2.1 Situacion Actual

El proceso actual de cotizaciones:
- **5 sistemas involucrados**: CRM, ERP, Excel, Email, Portal
- **Tiempo promedio**: 5 dias (vs 24h de competencia)
- **Errores de pricing**: 8% de cotizaciones con errores
- **Visibilidad**: Ninguna hasta que se completa

### 2.2 Problemas y Oportunidades

| Problema | Impacto | Oportunidad |
|----------|---------|-------------|
| Tiempo de respuesta | Perdida de 30% de deals | Ganar mas deals |
| Errores manuales | $500K en ajustes/ano | Eliminar costos |
| Falta visibilidad | Clientes frustrados | Mejorar experiencia |
| Datos fragmentados | Decisiones sin datos | Analytics |

### 2.3 Drivers de Cambio

1. **Competitivo**: Competidores responden en 24h
2. **Cliente**: NPS de ventas cayo 15 puntos
3. **Financiero**: Objetivo de reducir costos operativos 20%
4. **Estrategico**: OKR 2026 de "Experiencia digital diferenciada"

---

## 3. Vision de Arquitectura

### 3.1 Estado Futuro (Target)

> "Un sistema de cotizaciones end-to-end que permite a cualquier
> vendedor generar cotizaciones precisas en minutos, con aprobaciones
> automaticas para el 80% de los casos, y visibilidad total para
> clientes y gestion."

Caracteristicas clave:
- **Autoservicio**: Vendedor genera cotizacion sin ayuda de pricing
- **Automatico**: Precios calculados en tiempo real desde SAP
- **Inteligente**: Aprobaciones auto-rutadas segun reglas
- **Transparente**: Cliente ve estado en portal

### 3.2 Propuesta de Valor

| Stakeholder | Valor |
|-------------|-------|
| Ventas | Cotizar en minutos, no dias |
| Finanzas | Cero errores de pricing |
| Clientes | Visibilidad y respuesta rapida |
| Gestion | Dashboards de conversion y pipeline |

### 3.3 Principios Guia

1. **Integracion sobre duplicacion**: Usar datos de sistemas existentes
2. **Automatizacion por defecto**: Solo excepciones requieren humanos
3. **Mobile-first**: Vendedores en campo pueden cotizar
4. **Trazabilidad completa**: Auditoria de cada decision de pricing

---

## 4. Impacto y Beneficios

### 4.1 Beneficios de Negocio

| Beneficio | Metrica Actual | Metrica Objetivo | Valor |
|-----------|----------------|------------------|-------|
| Tiempo de cotizacion | 5 dias | <24 horas | +30% win rate |
| Errores de pricing | 8% | <0.5% | $475K/ano |
| Conversion | 20% | 35% | $2M revenue |

### 4.2 Beneficios Tecnicos

- Eliminacion de 3 integraciones batch manuales
- Reduccion de 5 sistemas a 3 (retiro de Excel y email)
- Plataforma para futuras automatizaciones de ventas

### 4.3 Metricas de Exito

| KPI | Objetivo | Medicion |
|-----|----------|----------|
| Tiempo de cotizacion p95 | <4 horas | Sistema |
| Tasa de error pricing | <0.5% | Auditoria mensual |
| Adopcion | >90% en 3 meses | Login por vendedor |
| NPS de proceso | +20 puntos | Encuesta trimestral |

---

## 5. Alcance

### 5.1 En Scope
- Proceso de cotizacion (crear, aprobar, enviar)
- Integracion Salesforce-SAP para pricing
- Motor de reglas de aprobacion
- Portal de cliente para seguimiento
- Dashboards de gestion

### 5.2 Fuera de Scope
- CRM core (Salesforce existente)
- ERP core (SAP existente)
- Proceso de facturacion
- Contratos y legales

### 5.3 Supuestos
- Salesforce y SAP permanecen como sistemas core
- APIs de SAP disponibles o se desarrollan
- Presupuesto aprobado para Q2 2026

### 5.4 Restricciones
- Go-live antes de Q4 2026
- Presupuesto maximo $500K
- No cambios a politica de pricing

---

## 6. Vista de Alto Nivel

### 6.1 Arquitectura Conceptual

```
+-------------+     +------------------+     +-------------+
|             |     |                  |     |             |
| Salesforce  |<--->| NUEVO: Motor de  |<--->|    SAP      |
| (CRM)       |     | Cotizaciones     |     |   (ERP)     |
|             |     |                  |     |             |
+-------------+     +--------+---------+     +-------------+
                             |
                    +--------+---------+
                    |                  |
               +----v----+        +----v----+
               | App     |        | Portal  |
               | Movil   |        | Cliente |
               +---------+        +---------+
```

### 6.2 Cambios Clave

| Componente | Baseline | Target |
|------------|----------|--------|
| Captura cotizacion | Salesforce + Excel | Salesforce integrado |
| Calculo pricing | Manual (3 personas) | Motor automatico |
| Aprobaciones | Email | Workflow en sistema |
| Comunicacion cliente | Email manual | Portal autoservicio |

---

## 7. Riesgos

| Riesgo | Probabilidad | Impacto | Mitigacion |
|--------|--------------|---------|------------|
| API SAP no disponible | Media | Alto | Validar en Fase B, alternativa batch |
| Resistencia de vendedores | Alta | Medio | Change management, champions |
| Complejidad reglas pricing | Media | Medio | Empezar con 80% de casos |
| Timeline agresivo | Media | Alto | Scope flexible, MVP primero |
```

## Consejos para un Vision Efectivo

| Hacer | Evitar |
|-------|--------|
| Enfocarse en valor de negocio | Detalles tecnicos profundos |
| Usar visuales simples | Diagramas complejos |
| Cuantificar beneficios | Promesas vagas |
| Ser realista en riesgos | Ocultar problemas |
| Lenguaje ejecutivo | Jerga tecnica |
