# Definicion del Alcance Organizacional

**Tiempo estimado**: 20 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Adaptacion del ADM (2.1.3)

---

## Por que importa este subtema?

"Enterprise" en TOGAF no significa automaticamente "toda la empresa". Definir el alcance correctamente es critico: muy amplio y el proyecto se vuelve inmanejable; muy estrecho y pierdes oportunidades de integracion y eficiencia.

Esta decision se toma en la **Fase Preliminar** y afecta todo el ciclo del ADM.

---

## Que es el Alcance Organizacional

El alcance define:
- **Que unidades organizacionales** estan incluidas
- **Que dominios de arquitectura** se abordaran
- **Que nivel de detalle** se producira
- **Que geografias** estan en scope

---

## Dimensiones del Alcance

### 1. Amplitud (Breadth)

Que parte de la organizacion esta cubierta?

```
+--------------------------------------------------+
|                 CORPORACION GLOBAL                |
|                                                  |
|  +------------+  +------------+  +------------+  |
|  | Division   |  | Division   |  | Division   |  |
|  | Americas   |  | EMEA       |  | APAC       |  |
|  |            |  |            |  |            |  |
|  | +--------+ |  | +--------+ |  | +--------+ |  |
|  | | Retail | |  | | Retail | |  | | Retail | |  |
|  | +--------+ |  | +--------+ |  | +--------+ |  |
|  | +--------+ |  | +--------+ |  | +--------+ |  |
|  | | B2B    | |  | | B2B    | |  | | B2B    | |  |
|  | +--------+ |  | +--------+ |  | +--------+ |  |
|  +------------+  +------------+  +------------+  |
+--------------------------------------------------+

Alcance posible:
- Enterprise: Toda la corporacion
- Segment: Division Americas
- Capability: Retail Americas
```

### 2. Profundidad (Depth)

Que dominios de arquitectura se incluyen?

```
Profundidad Completa:        Profundidad Parcial:
+-----------------------+    +-----------------------+
| Negocio        [X]    |    | Negocio        [ ]    |
| Datos          [X]    |    | Datos          [X]    |
| Aplicaciones   [X]    |    | Aplicaciones   [X]    |
| Tecnologia     [X]    |    | Tecnologia     [X]    |
+-----------------------+    +-----------------------+
```

### 3. Horizonte Temporal (Time Period)

Que horizonte cubre el target?

| Horizonte | Descripcion | Uso tipico |
|-----------|-------------|------------|
| Corto (6-12 meses) | Mejoras tacticas | Proyectos de capacidad |
| Medio (1-3 anos) | Transformaciones | Proyectos de segmento |
| Largo (3-5+ anos) | Vision estrategica | Arquitectura enterprise |

---

## Como Definir el Alcance

### Paso 1: Identificar Drivers

Que esta impulsando la necesidad de arquitectura?

| Driver | Alcance sugerido |
|--------|------------------|
| Fusion/Adquisicion | Enterprise (integracion completa) |
| Nueva linea de producto | Segment (area de negocio) |
| Modernizacion de sistema | Capability (sistema especifico) |
| Regulacion (GDPR, PCI) | Enterprise o Segment segun impacto |

### Paso 2: Evaluar Recursos

| Recurso | Alcance amplio | Alcance reducido |
|---------|----------------|------------------|
| Equipo grande (10+ arq) | Factible | Desperdicio |
| Equipo pequeno (1-3 arq) | Riesgoso | Apropiado |
| Presupuesto alto | Factible | Desperdicio |
| Tiempo limitado | Riesgoso | Apropiado |

### Paso 3: Considerar Interdependencias

```
Si el alcance es "Sistema CRM":
  - Depende de "Master Data de Clientes"?
  - Se integra con "Sistema de Facturacion"?
  - Usa "Plataforma de Integracion"?

Si hay dependencias fuertes, considerar expandir el alcance.
```

## Documento de Alcance

```markdown
# Definicion de Alcance - Proyecto [Nombre]

## Unidades Organizacionales en Scope
- [X] Division Retail Americas
- [X] Division Retail EMEA
- [ ] Division B2B (fuera de scope)
- [ ] Corporate (fuera de scope)

## Dominios de Arquitectura
- [X] Arquitectura de Negocio (procesos de venta)
- [X] Arquitectura de Datos (cliente, producto, transaccion)
- [X] Arquitectura de Aplicaciones (e-commerce, POS)
- [ ] Arquitectura Tecnologica (usar estandares existentes)

## Nivel de Detalle
- Negocio: Nivel L2 de capacidades
- Datos: Modelo conceptual + logico
- Aplicaciones: Portafolio + integraciones clave
- Tecnologia: Solo restricciones

## Horizonte Temporal
- Baseline: Estado actual (Q1 2026)
- Target: 18 meses (Q3 2027)
- Transiciones: 2 (Q4 2026, Q2 2027)

## Exclusiones Explicitas
- Sistemas de RRHH
- Infraestructura de red
- Seguridad fisica

## Interfaces con Fuera de Scope
- MDM Corporativo: Consumir, no modificar
- ERP (SAP): Integraciones definidas, arquitectura ERP no en scope
```

## Errores Comunes

| Error | Consecuencia | Prevencion |
|-------|--------------|------------|
| Alcance indefinido | Scope creep constante | Documentar y aprobar alcance |
| Alcance demasiado amplio | Proyecto interminable | Empezar con segmento, expandir |
| Omitir dependencias | Arquitectura incompleta | Mapear interfaces externas |
| Ignorar governance | Dudas sobre que esta en scope | Incluir proceso de cambio de alcance |
