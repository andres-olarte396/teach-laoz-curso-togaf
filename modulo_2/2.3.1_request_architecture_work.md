# Request for Architecture Work

**Tiempo estimado**: 20 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Establecimiento del Framework (2.2.3)

---

## Por que importa este subtema?

El Request for Architecture Work (RAW) es el documento que **inicia formalmente** un proyecto de arquitectura. Sin un RAW claro, los proyectos comienzan con ambiguedad sobre que problema resolver, quien es el sponsor, y que recursos estan disponibles.

Es la "orden de trabajo" que autoriza al equipo de arquitectura a comenzar.

---

## Que es el Request for Architecture Work

> "Un documento que proporciona un resumen del proyecto de desarrollo de arquitectura propuesto, incluyendo el alcance, objetivos, restricciones, y recursos."

**Analogia**: Es como una "solicitud de propuesta" interna donde el negocio pide al equipo de arquitectura que trabaje en un problema especifico.

---

## Quien lo Genera

| Actor | Rol |
|-------|-----|
| **Sponsor de negocio** | Solicita el trabajo, provee contexto de negocio |
| **CIO/CTO** | Aprueba recursos y prioridad |
| **Architecture Board** | Valida que es apropiado para arquitectura (vs proyecto operativo) |

## Ejemplo Practico

```markdown
# Request for Architecture Work

## 1. Informacion General
- **ID**: RAW-2026-003
- **Fecha**: 2026-02-15
- **Sponsor**: Maria Lopez, VP Comercial
- **Solicitante**: Carlos Ruiz, Director de Ventas

## 2. Contexto del Negocio

### 2.1 Situacion Actual
El proceso de cotizacion actual toma en promedio 5 dias debido a:
- Multiples sistemas desconectados (CRM, ERP, Excel)
- Aprobaciones manuales via email
- Falta de visibilidad del estado de cotizaciones

### 2.2 Drivers de Negocio
- Competidores responden cotizaciones en 24 horas
- Perdida estimada de $2M anuales por cotizaciones lentas
- Clientes clave han expresado frustracion

### 2.3 Objetivos de Negocio
- Reducir tiempo de cotizacion de 5 dias a 24 horas
- Aumentar tasa de conversion de cotizaciones de 20% a 35%
- Eliminar 100% de errores de pricing manual

## 3. Alcance Propuesto

### 3.1 En Scope
- Proceso de cotizacion end-to-end
- Integracion CRM-ERP para pricing
- Workflow de aprobaciones automatizado
- Portal de clientes para seguimiento

### 3.2 Fuera de Scope
- Sistema CRM core (usar Salesforce existente)
- Sistema ERP core (usar SAP existente)
- Proceso de facturacion

### 3.3 Restricciones
- Debe integrarse con Salesforce y SAP existentes
- Presupuesto de implementacion: max $500K
- Go-live requerido antes de Q4 2026

## 4. Stakeholders Clave
| Nombre | Rol | Interes |
|--------|-----|---------|
| Maria Lopez | VP Comercial | Sponsor, metricas de negocio |
| Carlos Ruiz | Dir Ventas | Usuario principal, requerimientos |
| Pedro Sanchez | Dir TI | Integracion tecnica |
| Ana Torres | Finanzas | Aprobacion de precios |

## 5. Recursos Propuestos

### 5.1 Presupuesto
- Arquitectura: $30,000
- Implementacion estimada: $400,000
- Contingencia: $70,000

### 5.2 Timeline Deseado
- Inicio arquitectura: 2026-03-01
- Arquitectura completada: 2026-04-15
- Implementacion completada: 2026-09-30

### 5.3 Equipo Sugerido
- Lead architect: 1 (tiempo completo)
- Solution architects: 2 (50% tiempo)
```

## Evaluacion del RAW

El Architecture Board debe evaluar:

| Criterio | Pregunta |
|----------|----------|
| **Claridad** | Esta claro que se pide? |
| **Alineacion** | Se alinea con estrategia? |
| **Factibilidad** | Es realista con los recursos? |
| **Valor** | El beneficio justifica el esfuerzo? |
| **Apropiado** | Es trabajo de arquitectura (vs proyecto operativo)? |

---

## Respuestas al RAW

| Respuesta | Cuando |
|-----------|--------|
| **Aprobado** | RAW claro, alineado, factible |
| **Aprobado con cambios** | Necesita ajustes menores de alcance o recursos |
| **Rechazado** | No alineado, no factible, o no es trabajo de arquitectura |
| **Diferido** | Buena idea pero no es el momento |
| **Mas informacion** | Falta contexto para decidir |

---

## Resumen

| Elemento | Proposito |
|----------|-----------|
| **RAW** | Documento que inicia proyecto de arquitectura |
| **Sponsor** | Quien solicita y financia |
| **Alcance** | Que esta dentro y fuera |
| **Restricciones** | Limites de tiempo, dinero, tecnologia |
| **Aprobacion** | Validacion por Architecture Board |
