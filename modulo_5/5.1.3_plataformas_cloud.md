# Plataformas y Cloud

**Tiempo estimado**: 30 minutos
**Nivel**: Intermedio-Avanzado
**Prerrequisitos**: Estandares Tecnologicos (5.1.2)

---

## Por que importa este subtema?

La decision de plataforma (on-premise, cloud, hibrido) es una de las mas impactantes en la arquitectura tecnologica. Afecta:

- Costos (CAPEX vs OPEX)
- Agilidad y time-to-market
- Seguridad y compliance
- Capacidades disponibles
- Skills requeridos

## Modelos de Despliegue

### On-Premise

```
+------------------------------------------+
|           DATA CENTER PROPIO              |
|                                          |
|  +--------+  +--------+  +--------+      |
|  |Servidor|  |Servidor|  |Servidor|      |
|  +--------+  +--------+  +--------+      |
|                                          |
|  +--------+  +--------+                  |
|  |Storage |  |Network |                  |
|  +--------+  +--------+                  |
|                                          |
+------------------------------------------+
        |
   Responsabilidad: 100% propia
```

**Caracteristicas**:
- Control total
- Mayor inversion inicial (CAPEX)
- Requiere equipo de infraestructura
- Capacidad fija, escalado manual

### Cloud Publico

```
+------------------------------------------+
|         CLOUD PROVIDER (AWS/Azure/GCP)    |
|                                          |
|  +--------+  +--------+  +--------+      |
|  | EC2    |  | RDS    |  | S3     |      |
|  +--------+  +--------+  +--------+      |
|                                          |
|  Gestionado por provider                 |
|  Pago por uso (OPEX)                     |
|                                          |
+------------------------------------------+
        |
   Responsabilidad: Compartida
```

**Caracteristicas**:
- Escalado automatico
- Pago por uso
- Servicios gestionados
- Menor control de infraestructura

### Nube Hibrida

```
+------------------+     +------------------+
|   ON-PREMISE     |     |   CLOUD PUBLICA  |
|                  |     |                  |
| +------+ +-----+ |     | +------+ +-----+ |
| | SAP  | |Datos| |<--->| | Apps | | AI  | |
| | Core | |Sens.| |     | | Web  | | ML  | |
| +------+ +-----+ |     | +------+ +-----+ |
|                  |     |                  |
+------------------+     +------------------+
         |                       |
         +-------+-------+-------+
                 |
            VPN / Direct Connect
```

**Caracteristicas**:
- Lo mejor de ambos mundos
- Complejidad de gestion
- Latencia entre ambientes
- Ideal para transicion

### Multi-Cloud

```
+-------------+  +-------------+  +-------------+
|    AWS      |  |   AZURE     |  |    GCP      |
| +---------+ |  | +---------+ |  | +---------+ |
| |Compute  | |  | |Identity | |  | |BigQuery | |
| |Storage  | |  | |Office365| |  | |AI/ML    | |
| +---------+ |  | +---------+ |  | +---------+ |
+-------------+  +-------------+  +-------------+
        \            |            /
         \           |           /
          +----------+---------+
          |   GESTION MULTI    |
          |   Terraform, K8s   |
          +--------------------+
```

**Caracteristicas**:
- Evita vendor lock-in
- Usa "best of breed" por servicio
- Alta complejidad operativa
- Requiere abstraccion

## Evaluacion de Estrategia Cloud

### 6 Rs de Migracion (AWS)

| Estrategia | Descripcion | Cuando usar |
|------------|-------------|-------------|
| **Rehost** (Lift & Shift) | Mover sin cambios | Migracion rapida, legacy |
| **Replatform** | Pequenas optimizaciones | Aprovechar servicios gestionados |
| **Repurchase** | Cambiar a SaaS | CRM, ERP, email |
| **Refactor** | Redisenar para cloud | Aplicaciones estrategicas |
| **Retire** | Eliminar | Sistemas obsoletos |
| **Retain** | Mantener on-prem | Compliance, latencia, costo |

### Matriz de Decision

```
                    VALOR ESTRATEGICO
                    Bajo           Alto
              +------------+------------+
         Alto | REPLATFORM | REFACTOR   |
COMPLEJIDAD   | o REHOST   | (Cloud-    |
TECNICA       |            | native)    |
              +------------+------------+
         Bajo | RETIRE o   | REPURCHASE |
              | RETAIN     | (SaaS)     |
              +------------+------------+
```

## Consideraciones Clave

### Seguridad en Cloud

| Aspecto | On-Premise | Cloud |
|---------|------------|-------|
| Perimetro | Firewall fisico | Security Groups, WAF |
| Identidad | Active Directory | IAM + Federation |
| Datos | Encriptacion manual | KMS, managed encryption |
| Compliance | Tu responsabilidad | Shared responsibility |

### Costos

```
ON-PREMISE:
- CAPEX alto inicial
- Costos fijos
- Capacidad ociosa
- Depreciacion

CLOUD:
- OPEX variable
- Pago por uso
- Optimizable
- Puede ser mas caro si no se gestiona
```

### Well-Architected Framework (AWS)

| Pilar | Descripcion |
|-------|-------------|
| Operational Excellence | Automatizacion, monitoreo, mejora continua |
| Security | Proteccion de datos e infraestructura |
| Reliability | Resiliencia y recuperacion |
| Performance Efficiency | Uso eficiente de recursos |
| Cost Optimization | Eliminar desperdicio |
| Sustainability | Reducir impacto ambiental |
