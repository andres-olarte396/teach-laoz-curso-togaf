# Estandares Tecnologicos

**Tiempo estimado**: 25 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Modelos de Referencia Tecnologica (5.1.1)

---

## Por que importa este subtema?

Los estandares tecnologicos reducen la complejidad y el costo de mantener la arquitectura. Sin estandares:

- Cada proyecto elige su propia tecnologia
- El conocimiento no se reutiliza
- Los costos de licencias se multiplican
- La integracion se complica

Los estandares no son restricciones arbitrarias: son decisiones arquitectonicas documentadas.

---

## Que es un Estandar Tecnologico

> "Una especificacion tecnologica adoptada por la organizacion que define productos, protocolos o practicas aprobadas para uso general."

**Tipos**:
- **Producto**: "Usamos PostgreSQL para bases de datos relacionales"
- **Protocolo**: "Las APIs usan REST sobre HTTPS"
- **Practica**: "El codigo se almacena en repositorios Git"

## Ciclo de Vida de Estandares

```
EMERGENTE     TACTICO      ESTRATEGICO    CONTENCION    RETIRO
    |            |              |              |           |
    v            v              v              v           v
+--------+   +--------+    +--------+     +--------+  +--------+
| Nueva  |   | Uso    |    | Uso    |     | Uso    |  | Migrar |
| tecnol |-->| limitado|-->| general|---->| limitado|->| fuera  |
| en eval|   | casos  |    | default|     | legacy |  |        |
+--------+   +--------+    +--------+     +--------+  +--------+

Tiempo -->
```

### Estados definidos

| Estado | Descripcion | Nuevos proyectos | Existentes |
|--------|-------------|------------------|------------|
| **Emergente** | En evaluacion, piloto | Con aprobacion | No |
| **Tactico** | Aprobado para casos especificos | Con justificacion | Permitido |
| **Estrategico** | Estandar por defecto | Recomendado | Migrar hacia |
| **Contencion** | No para nuevos usos | Prohibido | Mantener |
| **Retiro** | Debe eliminarse | Prohibido | Migrar fuera |

## Standards Information Base (SIB)

TOGAF sugiere mantener un repositorio de estandares:

```markdown
# Standards Information Base

## Metadata
- Version: 2.5
- Ultima actualizacion: 2026-01-28
- Aprobado por: Architecture Board
- Proxima revision: 2026-07-01

## Estructura
1. Estandares de Industria (externos)
2. Estandares Organizacionales (internos)
3. Excepciones Aprobadas
```

## Proceso de Gestion de Estandares

### Proponer nuevo estandar

```markdown
## Propuesta de Estandar: [Nombre]

### Informacion General
- **Proponente**: [Nombre, equipo]
- **Fecha**: [fecha]
- **Categoria TRM**: [categoria]

### Descripcion
[Que es y para que sirve]

### Justificacion
- Problema que resuelve
- Beneficios esperados
- Comparacion con alternativas

### Evaluacion
| Criterio | Score (1-5) | Notas |
|----------|-------------|-------|
| Madurez del producto | | |
| Soporte y comunidad | | |
| Fit con stack actual | | |
| Costo | | |
| Habilidades del equipo | | |

### Impacto
- Sistemas afectados
- Capacitacion requerida
- Costos de adopcion

### Recomendacion
[Emergente / Tactico / Estrategico]

### Aprobacion
| Rol | Nombre | Decision | Fecha |
|-----|--------|----------|-------|
| Architecture Board | | | |
```
