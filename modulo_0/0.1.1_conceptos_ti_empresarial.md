# Conceptos de TI Empresarial

**Tiempo estimado**: 30 minutos
**Nivel**: Basico
**Prerrequisitos**: Ninguno

---

## Por que importa este subtema?

Antes de abordar arquitectura empresarial, es fundamental entender como funcionan los sistemas de TI en las organizaciones. Este conocimiento base permite comprender el contexto donde se aplica TOGAF.

---

## Que es TI Empresarial

> "Tecnologia de la Informacion Empresarial abarca todos los sistemas, infraestructura, procesos y personas que soportan las operaciones de negocio mediante tecnologia."

**Componentes principales**:
- Infraestructura (servidores, redes, cloud)
- Aplicaciones (software de negocio)
- Datos (informacion del negocio)
- Personas (equipos de TI)
- Procesos (como se gestiona TI)

## Conceptos Clave de Infraestructura

### On-Premise vs Cloud

```markdown
## Comparativa

| Aspecto | On-Premise | Cloud |
|---------|------------|-------|
| Ubicacion | Datacenter propio | Proveedor externo |
| Inversion | CAPEX alto inicial | OPEX mensual |
| Escalabilidad | Limitada, planificada | Elastica, on-demand |
| Mantenimiento | Equipo interno | Proveedor |
| Control | Total | Parcial |

## Modelos de Cloud

| Modelo | Definicion | Ejemplos |
|--------|------------|----------|
| IaaS | Infraestructura como servicio | AWS EC2, Azure VMs |
| PaaS | Plataforma como servicio | Heroku, Azure App Service |
| SaaS | Software como servicio | Salesforce, Office 365 |
```

```
+----------+     +----------+     +----------+     +----------+
| PLANEAR  |---->| CONSTRUIR|---->| OPERAR   |---->| RETIRAR  |
+----------+     +----------+     +----------+     +----------+
     |                |                |                |

  Arquitectura    Deployment       Actualizaciones Archivo
```

### Estados de un Sistema

| Estado | Descripcion | Accion Tipica |
|--------|-------------|---------------|
| Nuevo | En desarrollo | Construir |
| Activo | En produccion | Mantener |
| Estable | Funcionando bien | Optimizar |
| Problematico | Fallos frecuentes | Evaluar reemplazo |
| Legacy | Tecnologia obsoleta | Planear retiro |
| Deprecated | Marcado para retiro | Migrar usuarios |
| Retired | Fuera de servicio | Archivar |

## Integracion de Sistemas

### Por que Integrar

```markdown
## Problemas sin Integracion

- Datos duplicados y inconsistentes
- Procesos manuales entre sistemas
- Falta de visibilidad end-to-end
- Errores por reingreso de datos
- Demoras en procesos de negocio
```

## Gobierno de TI

### Que es

> "Conjunto de politicas, procesos y estructuras que aseguran que TI soporte los objetivos del negocio de manera efectiva y controlada."

### Frameworks Comunes

| Framework | Enfoque | Uso |
|-----------|---------|-----|
| ITIL | Gestion de servicios TI | Operaciones |
| COBIT | Gobierno y control | Auditoria |
| TOGAF | Arquitectura empresarial | Estrategia |
| SAFe | Desarrollo agil a escala | Desarrollo |
