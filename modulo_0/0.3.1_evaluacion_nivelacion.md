## Seccion 1: TI Empresarial

### Preguntas

```markdown
## 1.1 Conceptos Basicos

1. Que tipo de sistema se usa principalmente para gestionar
   las finanzas, RRHH e inventario de una empresa?
   a) CRM
   b) ERP
   c) BI
   d) SCM

2. Cual es la diferencia principal entre IaaS y SaaS?
   a) IaaS es gratis, SaaS es de pago
   b) IaaS provee infraestructura, SaaS provee aplicaciones listas
   c) IaaS es on-premise, SaaS es cloud
   d) No hay diferencia

3. Que patron de integracion reduce la complejidad de
   conexiones punto a punto?
   a) File transfer
   b) API directa
   c) ESB (Enterprise Service Bus)
   d) Email

4. Que framework se enfoca en gestion de servicios de TI
   (operaciones)?
   a) TOGAF
   b) ITIL
   c) COBIT
   d) SAFe
```

### Respuestas

```markdown
1. b) ERP - Enterprise Resource Planning gestiona recursos core
2. b) IaaS provee infraestructura (VMs, storage), SaaS apps completas
3. c) ESB centraliza integraciones, reduce conexiones N*(N-1)/2
4. b) ITIL es para operaciones de servicios TI
```

## Seccion 2: Gestion de Proyectos

### Preguntas

```markdown
## 2.1 Fundamentos

1. Cuales son los tres elementos de la "triple restriccion"?
   a) Costo, Calidad, Riesgo
   b) Alcance, Tiempo, Costo
   c) Recursos, Presupuesto, Cronograma
   d) Inicio, Ejecucion, Cierre

2. En que metodologia se entregan incrementos funcionales
   en cada iteracion?
   a) Waterfall
   b) Agile
   c) PRINCE2
   d) Ninguna

3. Cual es el documento que autoriza formalmente el inicio
   de un proyecto?
   a) WBS
   b) Project Charter
   c) Cronograma
   d) Reporte de estado

4. En que fase del proyecto se define el alcance detallado
   y se crea el cronograma?
   a) Inicio
   b) Planeacion
   c) Ejecucion
   d) Cierre
```

### Respuestas

```markdown
1. b) Alcance, Tiempo, Costo - la triple restriccion clasica
2. b) Agile entrega incrementos en sprints/iteraciones
3. b) Project Charter autoriza formalmente el proyecto
4. b) Planeacion es donde se detalla alcance y cronograma
```

## Seccion 3: Modelado y Diagramacion

### Preguntas

```markdown
## 3.1 Notaciones

1. Que notacion se usa principalmente para modelar
   procesos de negocio?
   a) UML
   b) BPMN
   c) ERD
   d) ArchiMate

2. En un diagrama de secuencia, que representa el eje
   vertical?
   a) Componentes
   b) Tiempo
   c) Datos
   d) Usuarios

3. Cual de estos NO es una buena practica de diagramacion?
   a) Incluir leyenda cuando se usan simbolos no estandar
   b) Poner todo el detalle posible en un solo diagrama
   c) Mantener consistencia en simbolos
   d) Adaptar el nivel de detalle a la audiencia

4. ArchiMate organiza los elementos en capas. Cuales son
   las principales capas?
   a) Presentacion, Logica, Datos
   b) Frontend, Backend, Database
   c) Negocio, Aplicacion, Tecnologia
   d) Estrategia, Tactica, Operacion
```

### Respuestas

```markdown
1. b) BPMN - Business Process Model and Notation
2. b) Tiempo fluye de arriba hacia abajo en secuencia
3. b) Es mejor dividir en multiples diagramas que sobrecargar uno
4. c) Negocio, Aplicacion, Tecnologia (+ Estrategia y otros)
```

## Seccion 4: Estrategia de Negocio

### Preguntas

```markdown
## 4.1 Conceptos Estrategicos

1. Que analisis evalua Fortalezas, Oportunidades,
   Debilidades y Amenazas?
   a) Porter
   b) PESTEL
   c) FODA/SWOT
   d) BCG

2. En el Business Model Canvas, donde se describe
   "que problema resolvemos al cliente"?
   a) Segmentos de clientes
   b) Propuesta de valor
   c) Canales
   d) Recursos clave

3. Que significa la "A" en objetivos SMART?
   a) Agil
   b) Alcanzable
   c) Automatico
   d) Arquitectonico

4. Por que es importante la alineacion TI-Negocio?
   a) Para que TI tenga mas presupuesto
   b) Para que las inversiones en TI generen valor de negocio
   c) Para reducir el personal de TI
   d) Para comprar mas tecnologia
```

### Respuestas

```markdown
1. c) FODA/SWOT analiza interno (F/D) y externo (O/A)
2. b) Propuesta de valor describe que ofrecemos y que problema resuelve
3. b) Alcanzable - el objetivo debe ser realista
4. b) Alineacion asegura que TI soporte objetivos de negocio
```
