# Analisis de Brechas Consolidado

**Tiempo estimado**: 20 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Evaluacion de Work Packages (6.1.1)

---

## Por que importa este subtema?

Cada dominio de arquitectura (B, C, D) produce su propio gap analysis. La Fase E requiere consolidar estos gaps en una vista unificada para:

- Identificar dependencias entre gaps de diferentes dominios
- Evitar duplicacion de esfuerzos
- Priorizar de manera integral

---

## Proceso de Consolidacion

```
+---------------+     +---------------+     +---------------+
| Gap Analysis  |     | Gap Analysis  |     | Gap Analysis  |
| NEGOCIO       |     | DATOS/APPS    |     | TECNOLOGIA    |
| (Fase B)      |     | (Fase C)      |     | (Fase D)      |
+-------+-------+     +-------+-------+     +-------+-------+
        |                     |                     |
        +----------+----------+----------+----------+
                   |
                   v
        +----------+----------+
        | GAP ANALYSIS        |
        | CONSOLIDADO         |
        | (Fase E)            |
        +---------------------+
```

---

## Matriz Consolidada

```markdown
## Consolidated Gap Analysis

| ID | Gap | Dominio | Tipo | Prioridad | Dependencias |
|----|-----|---------|------|-----------|--------------|
| GAP-B-001 | Proceso de cotizacion manual | Negocio | Transformar | Alta | - |
| GAP-C-001 | Sin API de precios | Datos | Nuevo | Alta | - |
| GAP-C-002 | CRM legacy | Apps | Migrar | Alta | GAP-C-001 |
| GAP-D-001 | Servidores on-premise | Tech | Migrar | Media | GAP-C-002 |
| GAP-B-002 | Rol de capturista obsoleto | Negocio | Eliminar | Media | GAP-B-001 |
| GAP-C-003 | Datos duplicados | Datos | Resolver | Alta | GAP-C-002 |
```

## Reconciliacion de Conflictos

| Conflicto | Gap 1 | Gap 2 | Resolucion |
|-----------|-------|-------|------------|
| Mismo recurso | WP-001 (equipo A) | WP-004 (equipo A) | Secuenciar o agregar recursos |
| Prioridad diferente | Negocio: MDM Alta | Tech: MDM Media | Sponsor decide |
| Enfoque diferente | Build CRM | Buy CRM | Evaluar en Fase E |
