# Evaluacion de Work Packages

**Tiempo estimado**: 25 minutos
**Nivel**: Intermedio
**Prerrequisitos**: Modulo 5 completo

---

## Por que importa este subtema?

Los Work Packages son la unidad de planificacion que conecta la arquitectura con la implementacion. Transforman gaps abstractos en trabajo concreto que puede estimarse, asignarse y ejecutarse.

---

## Que es un Work Package

> "Un conjunto de actividades y entregables relacionados que pueden asignarse a un equipo para su implementacion."

**Caracteristicas**:
- Alcance definido y acotado
- Entregables claros
- Estimable en tiempo y costo
- Asignable a un equipo
- Tiene dependencias identificadas

---

## Derivando Work Packages de Gaps

```
GAP ANALYSIS
    |
    +-- Gap 1: Automatizar cotizaciones
    |       |
    |       +-- WP-001: Desarrollar motor de pricing
    |       +-- WP-002: Crear portal de cotizaciones
    |       +-- WP-003: Integrar con ERP
    |
    +-- Gap 2: Unificar datos de cliente
    |       |
    |       +-- WP-004: Implementar MDM
    |       +-- WP-005: Migrar datos de CRM legacy
    |
    +-- Gap 3: Modernizar infraestructura
            |
            +-- WP-006: Migrar a Kubernetes
            +-- WP-007: Implementar CI/CD
```

## Estructura de un Work Package

```markdown
# Work Package: WP-001

## Informacion General
| Campo | Valor |
|-------|-------|
| ID | WP-001 |
| Nombre | Desarrollar motor de pricing automatico |
| Gap relacionado | GAP-BUS-001: Automatizar cotizaciones |
| Prioridad | Alta |
| Estado | Planificado |

## Alcance
### Incluye
- Desarrollo de servicio de calculo de precios
- Integracion con catalogo de productos
- Reglas de descuento configurables
- API REST para consumo

### Excluye
- Integracion con ERP (WP-003)
- UI de configuracion de reglas (WP-008)

## Entregables
1. Servicio de pricing desplegado en EKS
2. Documentacion de API (OpenAPI)
3. Suite de pruebas automatizadas
4. Runbook de operaciones

## Estimaciones
| Concepto | Estimacion |
|----------|------------|
| Duracion | 8 semanas |
| Esfuerzo | 320 horas |
| Costo desarrollo | $40,000 |
| Costo infraestructura | $500/mes |

## Dependencias
| Tipo | Work Package | Descripcion |
|------|--------------|-------------|
| Depende de | WP-010 | Catalogo de productos debe existir |
| Bloquea a | WP-002 | Portal necesita API de pricing |

## Equipo
| Rol | Personas | Dedicacion |
|-----|----------|------------|
| Tech Lead | 1 | 50% |
| Backend Developer | 2 | 100% |
| QA Engineer | 1 | 50% |

## Riesgos
| Riesgo | Probabilidad | Impacto | Mitigacion |
|--------|--------------|---------|------------|
| Reglas de pricing complejas | Media | Alto | Workshops con pricing team |
| Integracion con catalogo | Baja | Medio | POC temprano |
```

## Agrupacion de Work Packages

### Por Proyecto

```
PROYECTO: Transformacion Comercial
|
+-- WP-001: Motor de pricing
+-- WP-002: Portal de cotizaciones
+-- WP-003: Integracion ERP
+-- WP-009: Capacitacion usuarios

PROYECTO: Modernizacion de Datos
|
+-- WP-004: Implementar MDM
+-- WP-005: Migracion CRM
+-- WP-010: Catalogo unificado
```

### Por Release/Iteracion

```
RELEASE 1.0 (Q2 2026)
+-- WP-010: Catalogo de productos
+-- WP-001: Motor de pricing (MVP)

RELEASE 2.0 (Q3 2026)
+-- WP-002: Portal de cotizaciones
+-- WP-003: Integracion ERP

RELEASE 3.0 (Q4 2026)
+-- WP-001: Motor de pricing (completo)
+-- WP-009: Capacitacion
```
