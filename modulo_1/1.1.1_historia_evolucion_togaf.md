# Historia y Evolucion de TOGAF

**Tiempo estimado**: 20 minutos
**Nivel**: Basico
**Prerrequisitos**: Ninguno

---

## Por que importa este subtema?

Conocer la historia de TOGAF te permite entender por que el framework esta estructurado como esta. Cada version resolvio problemas reales que las organizaciones enfrentaban. Entender esta evolucion te ayuda a apreciar las decisiones de diseno y a anticipar hacia donde se dirige el estandar.

---

## Origenes: El Contexto de los Anos 90

### El problema que TOGAF vino a resolver

En los anos 90, las grandes organizaciones enfrentaban un caos tecnologico:

- **Sistemas aislados**: Cada departamento compraba su propia tecnologia
- **Integraciones fragiles**: Conexiones punto a punto que se rompian constantemente
- **Duplicacion masiva**: Multiples sistemas haciendo lo mismo
- **Costos descontrolados**: Nadie sabia cuanto se gastaba en TI ni por que

El gobierno de Estados Unidos fue uno de los primeros en reconocer este problema. El Departamento de Defensa desarrollo TAFIM (Technical Architecture Framework for Information Management) como intento de estandarizar su infraestructura de TI.

### Nacimiento de TOGAF

En 1995, The Open Group (un consorcio de empresas tecnologicas) tomo TAFIM como base y creo TOGAF Version 1. El objetivo era crear un framework abierto, no propietario, que cualquier organizacion pudiera adoptar.

```
1995: TOGAF 1.0
  |
  +-- Basado en TAFIM del Departamento de Defensa de USA
  +-- Enfoque inicial: Arquitectura Tecnica solamente
  +-- Primer ADM rudimentario
```

## Evolucion por Versiones

### TOGAF 1-6 (1995-2001): Los Anos Formativos

| Version | Ano | Cambio Principal |
|---------|-----|------------------|
| 1.0 | 1995 | Framework inicial basado en TAFIM |
| 2.0 | 1996 | Refinamiento del ADM |
| 3.0 | 1997 | Mejoras en Technical Reference Model |
| 4.0 | 1998 | Expansion de guias |
| 5.0 | 1999 | Standards Information Base |
| 6.0 | 2001 | Consolidacion |

Durante esta epoca, TOGAF era principalmente un framework de **arquitectura tecnica**. No abordaba arquitectura de negocio, datos o aplicaciones de manera significativa.

### TOGAF 7 (2001): La Transicion

TOGAF 7 marco un punto de inflexion:

- Se renombro de "Technical Architecture Framework" a "Enterprise Architecture Framework"
- Se anadio soporte inicial para arquitectura de negocio
- El ADM se volvio mas robusto

```
TOGAF 6 y anteriores:        TOGAF 7 en adelante:

+------------------+         +----------------------+
| Arquitectura     |         | Arquitectura         |
| Tecnica          |         | EMPRESARIAL          |
+------------------+         |                      |
                             | +------------------+ |
                             | | Negocio          | |
                             | +------------------+ |
                             | | Datos            | |
                             | +------------------+ |
                             | | Aplicaciones     | |
                             | +------------------+ |
                             | | Tecnologia       | |
                             | +------------------+ |
                             +----------------------+
```

### TOGAF 8 (2002-2006): Maduracion

TOGAF 8 y sus actualizaciones (8.1, 8.1.1) consolidaron el framework:

- ADM completamente desarrollado con todas las fases actuales
- Enterprise Continuum formalizado
- Architecture Content Framework introducido
- Mayor enfoque en governance

Esta version establecio TOGAF como el estandar de facto en arquitectura empresarial.

### TOGAF 9 (2009): La Version Moderna

TOGAF 9.0 represento una reescritura significativa:

**Cambios estructurales**:
- Documento reorganizado en 7 partes
- Modularizacion del contenido
- Guias y tecnicas separadas del ADM core

**Nuevos conceptos**:
- Architecture Capability Framework
- Content Metamodel detallado
- Viewpoints y Views formalizados
- Business Transformation Readiness Assessment

**TOGAF 9.1 (2011)** y **TOGAF 9.2 (2018)** fueron actualizaciones incrementales:

| Version | Mejoras Principales |
|---------|---------------------|
| 9.1 | Clarificaciones, Business Architecture mejorada |
| 9.2 | Guias de seguridad, alineacion con otros estandares |

### TOGAF Standard (Actual)

Desde 2022, The Open Group renombro "TOGAF 9.2" como "TOGAF Standard" para reflejar su naturaleza de estandar vivo que se actualiza continuamente sin cambios de version mayor.

## Influencias y Relaciones con Otros Frameworks

TOGAF no existe en aislamiento. Ha sido influenciado por y ha influenciado otros frameworks:

### Frameworks que influyeron en TOGAF

- **TAFIM** (DoD): Base original
- **Zachman Framework**: Taxonomia de arquitectura
- **FEAF** (Federal Enterprise Architecture Framework): Practicas de gobierno

### Frameworks complementarios

- **ArchiMate**: Lenguaje de modelado (ahora tambien de The Open Group)
- **COBIT**: Governance de TI
- **ITIL**: Gestion de servicios de TI

### Competidores/Alternativas

- **Zachman**: Mas taxonomia que proceso
- **FEAF/DODAF**: Especificos para gobierno/defensa USA
- **Gartner EA**: Propietario

---

## Por que TOGAF domina el mercado

Segun estudios de la industria, mas del 80% de las organizaciones que adoptan un framework de arquitectura empresarial eligen TOGAF. Las razones:

1. **Abierto y gratuito**: El estandar es accesible sin costo
2. **Vendor-neutral**: No favorece ningun proveedor tecnologico
3. **Certificaciones establecidas**: Camino claro de profesionalizacion
4. **Comunidad activa**: Miles de practicantes comparten experiencias
5. **Adaptabilidad**: Funciona para empresas de cualquier tamano
